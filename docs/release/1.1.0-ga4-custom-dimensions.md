## Why this blocks meaningful analytics

The app already sends mode metadata on lifecycle events (`matchType`, `gameModeId`, `gameModeSection`, …). GA4 **cannot** break reports down by those parameters until each is registered in Firebase Console → **Analytics** → **Custom definitions**.

Allow 24–48h after registration for dimensions to populate.

---

## Event-scoped custom dimensions (register all)

Firebase Console → Custom definitions → **Create custom dimension** → Scope: **Event**.

| Parameter name | Display name | Priority events |
|----------------|--------------|-----------------|
| `matchType` | Match type | `turn_submitted`, `match_started`, `game_mode_played`, `match_completed` |
| `gameModeId` | Game mode ID | same |
| `gameModeSection` | Game mode section | same |
| `startSource` | Match start source | `match_started`, `game_mode_played`, `match_resumed` |
| `hasBot` | Has bot opponent | lifecycle + `turn_submitted` |
| `botDifficulty` | Bot difficulty | lifecycle |
| `configStartScore` | X01 start score | X01 starts / turns |
| `configCheckoutMode` | Checkout mode | X01 starts / turns |
| `participantCount` | Participant count | lifecycle |
| `status` | Match status | `turn_submitted`, completion events |
| `durationSeconds` | Match duration (s) | `match_completed`, `match_forfeited`, `game_mode_forfeited` |

| `contentSizeCategory` | Dynamic type bucket | `client_environment_changed` |

| `isLowPowerModeEnabled` | Low power mode | `client_environment_changed` |

## Verification after registration

1. Install **TestFlight Release** build (real `GoogleService-Info.plist`).
2. Play one **Raid** match and one **party** match to completion.
3. Firebase Console → **Analytics** → **Realtime** or **DebugView**:
   - Confirm `game_mode_played` / `match_completed` include `gameModeId`.
4. After 24–48h, run an Exploration:
   - `game_mode_played` broken down by **Game mode ID** custom dimension.
   - Funnel: `match_setup_start` → `game_mode_played` → `match_completed` by mode.
