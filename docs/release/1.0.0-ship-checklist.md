## §2 App Review hardening (P0)

*Reviewer-visible leaks and metadata mismatches are higher risk than compiled hidden code.*  
Plan: [`lean-1.0-app-review-hardening-plan.md`](lean-1.0-app-review-hardening-plan.md)

### Hidden surface (Release build, no launch args)

- [ ] **Four tabs only** — Play, Players, Activity, Settings — **no Modes tab**
- [ ] Play setup mode picker: **X01 + Cricket only** — no party/practice teaser or “Coming Soon” rows
- [ ] **No path** to Baseball, Killer, Shanghai gameplay
- [ ] Player detail: **no** Export, **no** Training Partner (custom bots **are** OK)
- [ ] Activity filters: All / X01 / Cricket only
- [ ] Rules / onboarding rules: X01 + Cricket only

### Party-mode resume leak (engineering)

- [x] `ProductSurface.isMatchTypeReachable(_:)` gates party match types when `showsPartyModes == false`
- [x] Play home, Activity resume, History resume, deep links **cannot** open hidden match screens
- [x] With seeded active party match in DB: Play home shows setup, **not** resume into party screen
- [x] Unit tests: `ProductSurfaceTests`, `PlayHomeViewModelTests`, `AppRouteRouterTests`

### Lean mode picker (engineering)

- [x] `GameModeCatalog.playSetupPickerSections()` returns **standard section only** when party hidden
- [x] `GameModeCatalogTests` + `Lean1_0SmokeUITests` updated for no teaser sections in lean 1.0

### Custom bots (shipped in 1.0)

- [ ] Play setup + Players → Add Bot: preset + custom creation works
- [ ] Custom bot completes X01 and Cricket match

## §7 Accessibility & App Store Nutrition Labels (P0)

- VoiceOver
- Dark Interface
- Sufficient Contrast
- Larger Text
